# tests/

This is the real spec — the executable one.

In KISpec you don't write a formal requirements document. You write tests that
pin down the requirement, and you let them fail until the loop makes them pass.
Tests persist, catch regressions and don't lie; prose rots the moment the code
moves ahead of it.

Put your tests here, in whatever framework the project uses. There's no rule
about structure — only that a requirement worth stating is worth a test.
